## Toolchain

- Use npm (`package-lock.json`) with Node.js `>=22.12.0`; this is an Astro 7 static site.
- The root `README.md` is unchanged starter documentation and does not describe the current site architecture.

## Development

- Install with `npm install` and verify repository-wide changes with `npm run build`.
- No test, lint, formatter, or dedicated typecheck scripts are configured.
- Start the dev server in background mode with `npm run astro -- dev --background`.
- Manage it with `npm run astro -- dev status`, `npm run astro -- dev logs`, and `npm run astro -- dev stop`.

## Architecture

- `src/layouts/BaseLayout.astro` owns global CSS/font imports, the shared sidebar/footer, and Vercel analytics; use it for Astro pages.
- Markdown routes in `src/pages/` use `src/layouts/MarkdownLayout.astro` and require a `title` frontmatter field.
- Blog posts live in `src/content/blog/`; their frontmatter must satisfy `src/content.config.ts`. Routes are generated by `src/pages/blog/[...slug].astro` from each post's content ID.
- Font CSS in `src/fonts/` is generated by `cn-font-split` and references matching `.woff2` assets under `public/fonts/`; do not edit the generated CSS manually or move assets independently.
